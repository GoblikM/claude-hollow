# Office – AI Agent Orchestration System

A system for managing Claude Code AI agents during software development. Combines GTD task management with automated agent orchestration — no containers required.

## How it works

You start a feature by running `feature.sh`. Claude opens as an **orchestrator** for that feature and handles everything automatically:

1. Breaks down the request into concrete tasks
2. Creates `task.md` files in the GTD folder structure
3. Launches subagents (`@task-agent`, `@code-reviewer`, `@tester`) directly in the Claude session
4. Reviews the agent's diff after completion
5. Merges the task branch into the feature branch if accepted
6. Archives the completed task
7. Writes documentation to `docs/` summarizing what was implemented
8. Notifies you when the feature is ready for a merge request

```bash
cd office/
./scripts/feature.sh my-feature --project ../my-project
# Claude opens as orchestrator — describe what you want to implement
```

The orchestrator **never writes code itself** — that is exclusively the role of the agents.

## Structure

```
office/
├── CLAUDE.md                   # Orchestrator context (read by Claude on startup)
├── features/
│   ├── _templates/             # Templates: task, issue, inbox, feature docs
│   └── <feature-name>/         # Local only — not tracked in git
│       ├── CLAUDE.md           # Feature-specific orchestrator context (generated by feature.sh)
│       ├── workspace/          # Git worktree of the feature branch (project repo)
│       ├── tasks/              # Ready-to-run tasks
│       │   └── done/           # Completed tasks (archive)
│       ├── blocked/            # Tasks waiting on external dependencies
│       ├── icebox/             # Deliberately deferred items
│       └── docs/               # Feature documentation written after completion
├── inbox/                      # Quick capture — local only, not tracked in git
└── scripts/
    ├── office.sh               # Interactive entry point — manage projects and start features
    ├── feature.sh              # Initialize a feature workspace and git worktree
    ├── lib.sh                  # Shared utilities (branch detection, slugify, etc.)
    ├── task-done.sh            # Move a completed task to done/
    ├── cc.sh                   # (Advanced) Launch a task agent in an isolated git clone
    └── cleanup-clone.sh        # (Advanced) Remove a task clone after merge
```

## Usage

### Interactive (recommended)

```bash
./scripts/office.sh
```

Shows a menu to add/create projects and start features. Projects are stored in a local registry (`.projects`, gitignored) so you don't have to pass `--project` every time.

### Direct

```bash
# New feature (requires --project)
./scripts/feature.sh my-feature --project /path/to/project

# Reopen existing feature (project path loaded from CLAUDE.md)
./scripts/feature.sh my-feature

# Start feature branch from a specific branch
./scripts/feature.sh my-feature --project /path/to/project --from some-branch

# Delete a feature
./scripts/feature.sh -D my-feature
```

## Agent pipeline

Each task goes through three subagents in order, called directly in the Claude orchestrator session:

| Agent | Role | Trigger |
|-------|------|---------|
| `@task-agent` | Implements changes, commits to `task/<slug>` branch | First step |
| `@code-reviewer` | Reviews diff, verifies AC and conventions | After task-agent |
| `@tester` | Runs tests | After APPROVED |

## GTD rules

| Folder | Meaning |
|--------|---------|
| `tasks/` | Actionable, ready to run |
| `tasks/done/` | Completed (archive) |
| `blocked/` | Waiting on something external |
| `icebox/` | Intentionally deferred |
| `inbox/` | Unprocessed — review and move ASAP |

## Local data

`features/` and `inbox/` are gitignored — task data stays local. Only the tool itself (scripts, templates, agent definitions) is tracked in git. This means office works for any project without polluting the repo with project-specific task data.

## Requirements

- [Claude Code CLI](https://claude.ai/code) — `claude` available in PATH
- Git
- A git repository to work on (passed via `--project`)
